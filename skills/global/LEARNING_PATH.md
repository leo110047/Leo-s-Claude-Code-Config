# Global Skills Learning Path

這份文件給第一次接觸 goldband global skills 的人，重點是學習順序，不是操作細節。

## 第 1 階段：先建立底線

先熟悉：

- `evidence-based-coding`
- `systematic-debugging`
- `file-search`

目標：
- 不憑印象回答
- 知道 bug 要先重現、收證據、再修
- 習慣先建 codebase map

## 第 2 階段：學會規劃與審查

加入：

- `planning-workflow`
- `testing-strategy`
- `code-review-skill`

目標：
- 寫得出可執行 plan
- 知道怎麼設計 coverage 與 regression test
- 知道 review 要先看 correctness、risk、missing tests

## 第 3 階段：學會做方向判斷

加入：

- `backend-patterns`
- `api-design`
- `security-checklist`
- `performance-optimization`

目標：
- recommendation 不是只講 tradeoff
- 要能交代 assumptions、failure modes、switch criteria、best alternative

## 第 4 階段：維護 skill 與流程本身

加入：

- `decision-log`
- `new-skill-scaffold`
- `skill-developer`
- `claude-config-verification`

目標：
- 能維護 reusable workflow
- 能驗證 hooks / plugin / config surface
- 能把決策沉澱成 ADR，而不是只留在對話裡
